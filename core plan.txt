This platform is a **rule-based, admin-controlled opportunity system** focused on **Jobs, Internships, and Walk-ins** for students and fresh graduates.

The product exists to solve a single critical issue:

> *Students miss real opportunities because job information is scattered across hundreds of groups, channels, and pages.*

This platform centralizes, filters, validates, and tracks opportunities in a **strict, stage-driven flow**. There are **no recruiters**, **no automation engines**, and **no intelligent assistants** inside the system.

Every feature is built using **clear rules, explicit conditions, and manual verification**.

## ğŸ—ï¸ STAGE 12: SYSTEM ARCHITECTURE (NON-NEGOTIABLE)

This platform **must** use a **separate backend service**.

### Explicitly Disallowed

* Frontend-only backend logic
* Next.js API routes for core platform rules

### Approved Architecture

* Frontend: Next.js (Web + PWA)
* Backend: Node.js (Express) or FastAPI
* Database: PostgreSQL
* Authentication: JWT
* Background Jobs: Server-based scheduler (cron)

### Responsibility Split

**Frontend**

* UI rendering
* Client routing
* Token handling
* API consumption

**Backend**

* Business rule enforcement
* Eligibility filtering
* Expiry automation
* Admin-only operations
* User activity tracking
* Reporting and moderation

### Architectural Rule

> **Next.js renders the product.**
> **The backend runs the platform.**
